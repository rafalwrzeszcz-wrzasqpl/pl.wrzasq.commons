General information

This library contains set of small integration/wrapper classes. The point is to cover common integration cases (especially for Lambda development), without bloating the code and dependencies. Main purpose is to separate Lambda entry point from the logic. This allows making thin, lightweight entry points that interact with implementation, which you can encapsulate in the testable library.

Logging

As usual in Chillout Development projects, SLF4J is used. For using with AWS lambda we recommend using io.symphonia:lambda-logging as a backend.
